动态规划

发表于 2021-11-09 | 分类于 动态规划

Dynamic Programming

简称DP

称为记忆化搜索

最大特点是：

  • 用“空间”换“时间”
  • 不需要重复计算已经计算的值

核心解决的思路及问题：

  • dp数组及下标的含义
  • 递推公式
  • dp数组如何初始化
  • 遍历顺序（比较考究）
    • 如：从后往前遍历
  • （出现问题时）打印dp数组判断错误

明确「状态」 -> 定义 dp 数组/函数的含义 -> 明确「选择」-> 明确 base case

递推公式一般类型：

  • 斐波那契型F(n) = F(n - 1) + F(n - 2)

    这种只需管理两个值F(n - 1) 和 F(n - 2)来递推出F(n)

    如：

    • 爬楼梯问题

应用情况：

  • 股票问题
  • 背包问题
  • 递归函数或一般函数循环中出现大量重复计算的结点
  • 寻路
  • 子序列问题

动态规划一般思路：

  • 在程序中出现大量重复计算的结点时，尝试用动态规划
  • 用哈希表来记录已计算的结点
  • 再次遇到计算重复的节点时，直接查表，避免重复计算

509.斐波那契数列

动态规划新手题

初始化为fib(0)=0; fib(1)=1

而fib(2) = fib(1) + fib(0)

不难看出当n>1时，fib 的递推公式就是F(n) = F(n - 1) + F(n - 2)

而这里只需要管理两个值 F(n - 1) 和 F(n - 2) 就可以递推，没必要记录整个递推数组

心得：

  • 真没发现斐波那契数列还能有其他方法，而且比递归快多了

70.爬楼梯

题目：假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

通过枚举，不难发现

  • 爬到第一层楼梯有一种方法，爬到二层楼梯有两种方法
  • 那么第一层楼梯再跨两步就到第三层，第二层楼梯再跨一步就到第三层
  • 所以到第三层楼梯的状态可以由到第二层楼梯和到第一层楼梯状态推导出来
  • 那么就可以想到动态规划了
  • 其递推公式有点类似斐波那契数列的动态规划

这里有两种方法：

  • 记录整个递推数组
  • 不记录整个递推数组

相对来说，不记录整个递推数组占内存会少一点，虽然力扣上的内存消耗显示都一样是5.8MB（→__→）

心得：

  • 做的那时候，没有枚举，所以我没发现原来跟斐波那契数列的递推公式是一样的
  • 这让我意识到了读到题的前期，一定要枚举，不然发现不了规律

思路：

  • “备忘录”思想

  • dp数组：记录从(0,0)到各点有多少条不同的路径

  • 递推公式：dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

    dp[i][j]只能从横向和纵向推出

  • dp数组初始化：初始化 (i, 0) 和 (0, j) 的点全为1，从 (0, 0) 的位置到 (i, 0) 和 (0, j) 的路径都只有一条

心得：

一开始我的思路是像那种走迷宫思维，但发现没头绪，后来看思路，发现原来可以从dp[i - 1][j] 和 dp[i][j - 1]推出来，真想不到。看Carl哥题解还可以用一维数组，也用数论方法，排列组合，因为机器到底右下角，向下几步，向右几步都是固定的，比如，m=3, n=2，我们只要向下 1 步，向右 2 步就一定能到达终点。所以有 $C_{m+n-2}^{m-1}$或$C_{m+n-2}^{n-1}$，因为$C_m^n=C_m^{m-n}$。具体怎么实现还是个问题。

343.整数拆分

给定一个正整数 n，将其拆分为至少两个正整数的和，并使这些整数的乘积最大化。 返回你可以获得的最大乘积。

示例 1:

示例 2:

思路：

  • “备忘录”思维 + 最值

  • dp数组：记录每个值对应所乘的最大值

  • 递推公式：找最值嘛

    dp[i] = max(dp[i], max((i - j) * j, j * dp[i - j]))

  • 两层循环：

    • 外层循环：i 表示当前所求的最大乘值
    • 内层循环：j 表示 [2 , i-1) 一个一个相乘判断取最值

心得：

一开始没思路，我觉得应该是跟其他题一样需要备忘录，于是我列出[1, 10] 的情况，观察寻找规律，结果还是没发现，看了思路，原来需要结合最值，而且我列到9的时候，我发现9=4+5，4*5=20，但是把 5 分解成2+3，可以4*3*2=24，于是我想到了可以”查备忘录”
