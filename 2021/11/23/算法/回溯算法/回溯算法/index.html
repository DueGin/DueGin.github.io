回溯算法

发表于 2021-11-23 | 分类于 搜索算法

Backtracking

回溯法，一般可以解决如下几种问题：

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N皇后，解数独等等

回溯法效率本身很低，而且回溯函数是递归函数

因为回溯的本质是穷举，穷举所有可能，然后选出我们想要的答案，如果想让回溯法高效一些，可以加一些剪枝的操作，但也改不了回溯法就是穷举的本质。

回溯法解决的问题都可以抽象为决策树

因为回溯法解决的都是在集合中递归查找子集，集合的大小就构成了树的宽度，递归的深度，都构成的树的深度。

递归就要有终止条件，所以必然是一颗高度有限的树（N叉树）。

解决⼀个回溯问题，实际上就是⼀个决策树的遍历过程。你只需要思考 3 个问题：

1、路径：也就是已经做出的选择。 （记录你已经做过的选择）

2、选择列表：也就是你当前可以做的选择。

3、结束条件：也就是到达决策树底层，⽆法再做选择的条件。 （在这⾥就是选择列表为空的时候）

回溯可以剪枝来优化：

n - (k - path.size()) + 1这里表示的是，位数为k，而组合数是递增的，例如n=10，k=3，当层级在第一级时，path.size()=0，n - (k - path.size()) + 1=8，那从左至右只要遍历到 8 即可，因为只能组合成 [8, 9, 10] ，而 8 以后，9和10根本组合不了，根据这个思想，直接推广到所有层级

心得：

第一次做回溯题，首先在知道有模板的前提下，不会用模板，对于结束条件不太清楚，还有树的深度和树的宽度，刚开始还不知道深度用递归遍历，宽度用for遍历
