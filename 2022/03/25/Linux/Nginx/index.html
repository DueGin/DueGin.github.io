Nginx

Nginx(“engine x”)是一款是由俄罗斯的程序设计师 Igor Sysoev 所开发高性能的 Web 和 反向代理 服务器，也是一个 IMAP/POP3/SMTP 代理服务器。

在高连接并发的情况下，Nginx是Apache服务器不错的替代品。

所以我也不懂为什么 Nginx 也要制裁俄罗斯

1、Nginx常用功能

Http代理，反向代理：作为web服务器最常用的功能之一，尤其是反向代理。

Nginx在做反向代理时，提供性能稳定，并且能够提供配置灵活的转发功能。Nginx可以根据不同的正则匹配，采取不同的转发策略，比如图片文件结尾的走文件服务器，动态页面走web服务器，只要你正则写的没问题，又有相对应的服务器解决方案，你就可以随心所欲的玩。并且Nginx对返回结果进行错误页跳转，异常判断等。如果被分发的服务器存在异常，他可以将请求重新转发给另外一台服务器，然后自动去除异常服务器。

2、负载均衡

Nginx提供的负载均衡策略有2种：内置策略和扩展策略。内置策略为轮询，加权轮询，Ip hash。扩展策略，就天马行空，只有你想不到的没有他做不到的啦，你可以参照所有的负载均衡算法，给他一一找出来做下实现。

上3个图，理解这三种负载均衡算法的实现

Ip hash算法，对客户端请求的ip进行hash操作，然后根据hash结果将同一个客户端ip的请求分发给同一台服务器进行处理，可以解决session不共享的问题。

3、web缓存

Nginx可以对不同的文件做不同的缓存处理，配置灵活，并且支持 FastCGI_Cache，主要用于对 FastCGI 的动态程序进行缓存。配合着第三方的 ngx_cache_purge，对制定的 URL 缓存内容可以的进行增删管理。

优点

优点：

  • 高并发、高性能

    往往高并发只需要我们对每个连接所使用的内存尽量的少就可以达到，同时达到高性能，需要其架构非常的好

  • 可扩展性好

    主要体现在其模块化设计非常好，模块化设计非常的稳定，使得Nginx的第三方模块生态圈非常的丰富

  • 高可靠性

    持续不间断的运行数几年，疍机时间几乎以秒来计

  • 热部署

    在不停止 Nginx 的情况下升级 Nginx

  • BSD许可证

Nginx的组成

  • Nginx 二进制可执行文件：汽车组件
  • Nginx.conf 配置文件：驾驶者
  • access.log 访问日志：行车记录仪
  • error.log 错误日志：黑匣子

版本选择

版本选择：

  • Nginx社区版（stable version稳定版）
    • 大部分需求选择
  • OpenResty社区版
    • 更适用于开发 api服务器 和 web 防火墙

配置

语法：

指令块有无名字是由提供这个指令的nginx模块决定的，它可以决定指令块后面的有无参数，这些参数还可以通过类似shell函数的取参形式使用，即$1取第一个参数 …

http配置的指令块

  • http
    • 此指令块全部是由http指令去解析执行的
  • server
    • 对一个或一组域名
  • location
    • url 表达式
  • upstream
    • 上游服务，当我们 nginx 需要与企业内网交互的时候，如 Django ，我们可以定义一个upstream

nginx命令参数

nginx：

  • -h：帮助文档
  • -c：指定配置文件
  • -g：指定配置指令
  • -p：指定运行目录
  • -s：发送信号
    • stop：立刻停止服务
    • quit：优雅的停止服务，在退出前完成已经接受的连接请求
    • reload：重载配置文件（不会停止服务）
      • 执行此命令后，master进程会生成新的worker进程，且新进来的请求不允许旧worker进程接收请求了，转而给新的worker进程处理，当旧的worker进程处理完手头上的请求时，他则会退出
    • reopen：重新开始记录日志文件
  • -t 或 -T：测试配置文件是否有语法错误
  • -v：nginx版本信息
  • -V：configure脚本的编译信息

nginx工作流程

热部署

  1. 先备份一下旧的二进制 nginx 文件

  2. 然后将新的二进制 nginx 文件替换一下这个文件

  3. 向nginx的master进程发送一个 USR2信号：kill -USR2 UID（需填）

    虽然老的master和worker进程仍在运行，但老的master进程会生成一个新的master进程，而新的master进程会生成新的worker进程，实现平滑过渡

  4. 向nginx老的master进程发送WINCH信号，告诉它关闭所有旧的worker进程：kill -WINCH UID

    而老的master进程不会自己退出，可以通过reload命令重新拉起旧的worker进程，方便回滚

日志切割

拿taohui_access.log举例：

  2. 然后执行 nginx 命令：nginx -s reopen

gzip：压缩（提升传输效率）

  • gzip_comp_level：压缩级别
