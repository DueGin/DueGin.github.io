开始

  1. 创建一个Django项目

    创建完项目之后，会在当前目录下生成一个名为mysite的文件

    其文件结构为：

    • mysite/
      • manage.py
      • mysite/
        • __init__.py
        • settings.py （配置文件）
        • urls.py （全局总路由）
        • asgi.py
        • wsgi.py

    打开settings.py

    • 找到 ALLOWED_HOSTS=[]，修改成 ALLOWED_HOSTS=["自己的服务器的公网IP"]，否则无法通过自己的公网IP来访问
    • 找到 TIME_ZONE，更改时区为 'Asia/Shanghai'

    urls.py作用解析：所谓路由即是网址

    所有路由 include 的路径都是基于一级mysite目录的

    创建超级用户：python3 manage.py createsuperuser

    修改用户密码：python3 manage.py changepassword 用户名

  2. 启动项目

    在一级mysite目录下运行

    端口号自己随便填，后续访问网站时加上同样的端口号即可

  3. 创建分支

    1. 在一级mysite目录下运行

      python3 manage.py startapp game(app名字)

    在当前目录下会生成一个同名文件夹，其文件结构：

    • game/
      • __init__.py
      • admin.py （存管理员页面）
      • apps.py
      • migrations/ （存数据库）
        • __init__.py
      • models.py （数据类库）
      • tests.py
      • views.py （写函数返回网页html源码的）

    1. 创建完分支后，需要更新一下数据库 python3 manage.py migrate，然后再创建管理员账号 python3 manage.py createsuperuser ，然后就可以用 网站/admin 进入管理员页面了

    2. 建议在分支下也创建一个 urls.py，方便管理分支的总路由

    3. 注册 app，在 settings.py 中找到 INSTALLED_APPS，将 apps.py 中的类名添加到其中，注意路径同样是基于项目根目录，如：app名为game，则在 settings.py 中应添加 game.apps.GameConfig

    4. 设置静态文件的地址

      1. 在 settings.py 中找到 STATIC_URL，在这一条的上一行加入一个STATIC_ROOT = os.path.join(BASE_DIR, 'static') ，同时要记得 import os，os.path.join() 是将路径合并在一起。这一步代表说明我们的静态文件会放到这个目录下。BASE_DIR 在本文件的上面已声明，表示当前项目的根目录

      2. 还需写上 MEDIA_ROOT = os.path.join(BASE_DIR, 'media') 和 MEIDA_URL = /media/

        一般来说，static 存开发的文件，media 存用户的文件

    5. 文件分配：静态文件存在 static 里面，而 static 可以分为：

      • static/
        • css/
          • game.css
        • js/
          • dist/
            • game.js（汇总js）
          • src/
            • …（js源文件）
        • image/
          • …
      • audio/
         -   ...

      css文件一般只需一个，而 js 文件则需要分成多个，这样不容易乱，而且每个子模块逻辑相互独立，便于后期扩展和开发

      最好将复杂的，业务多的细分再细分，但是 js 文件需要打包汇总到 dist/game.js 中

      {% load static %} 是 Django 里的自动查询文件地址，自动定位到静态文件 static 的文件夹。

文件结构：

项目系统设计

  • menu：菜单页面
  • playground：游戏界面
  • settings：设置界面

项目文件结构

  • game目录：

    • templates目录：管理html文件

    • urls目录：管理路由，即链接与函数的对应关系

    • views目录：管理http函数

    • models目录：管理数据库数据

    • static目录：管理静态文件，比如：

      • css：对象的格式，比如位置、长宽、颜色、背景、字体大小等
      • js：对象的逻辑，比如对象的创建与销毁、事件函数、移动、变色等
      • image：图片
      • audio：声音
      • …

    • consumers目录：管理websocket函数
