fork

创建新的子进程

fork() 的一个奇妙之处就是它仅仅被调用一次，却能够返回两次，它可能有三种不同的返回值：

  • （>0）在父进程中，fork返回新创建子进程的进程ID；
  • （=0）在子进程中，fork返回0；
  • （<0）如果出现错误，fork返回一个负值；

fork() 出错可能有两种原因：

  • 当前的进程数已经达到了系统规定的上限，这时 errno 的值被设置为 EAGAIN。

  • 系统内存不足，这时 errno 的值被设置为 ENOMEM。

exec族

在进程中执行新的程序，且执行成功后不会返回

头文件：unistd.h

  • path 可执行程序的路径

  • arg 如果这个可执行程序需要传参，罗列参数，后面以 NULL 或以 (char *)0 为结束标志

  • file 可执行文件的文件名

  • envp[] 环境变量

例如：

先定义一个指针数组：char *argv[] = {"ls", "-l", (char *)0};

用 execv 调用 ls ：execv("/bin/ls", argv)

用 execvp 调用 ls ：execvp("ls", argv);

与一般情况不同，exec 函数族的函数执行成功后不会返回，因为调用进程的实体，包括代码段，数据段和堆栈等都已经被新的内容取代，只留下进程 ID 等一些表面上的信息仍保持原样。

wait()

阻塞本进程，等待某个子进程退出后才继续执行

系统调用API原型：

  • pid_t wait(int *status);

  • pid_t waitpid(pid_t pid, int *status, int options);

  • 进程一旦调用了 wait()，就立即阻塞自己，由 wait() 自动分析是否当前进程的某个子进程已经退出，如果让它找到了这样一个已经变成僵尸态的子进程，wait() 就会收集这个子进程的信息，并把它彻底销毁后返回；如果没有找到这样一个子进程，wait() 就会一直阻塞在这里，直到有一个出现为止。

  • wait() 要与 fork() 配套出现，如果在使用 fork() 之前调用 wait()，wait() 的返回值则为 -1，正常情况下 wait() 的返回值为子进程的 PID

  • 当父进程没有使用 wait() 函数等待已终止的子进程时，子进程就会进入一种无父进程清理自己尸体的状态，此时的子进程就是僵尸进程，不能在内核中清理尸体的情况

参数说明：

  • status

    用来保存被清理进程退出时的一些状态。如果不需要知道这个子进程如何死掉的，那也可以设定为 NULL

  • pid

    指定子进程的 ID。

    • pid > 0：等待进程ID为 pid 的进程退出
    • pid == -1：等待所有子进程退出，与 wait() 一样
    • pid == 0：等待同一个进程组的所有子进程退出
    • pid < -1：等待指定进程组的所有子进程退出，这个进程组的 ID 等于 pid 的绝对值

  • options

    额外选项控制 waitpid ，只支持 WNOHANG 、 WUNTRACED 和 0 三个常数，可以用 | 运算符把它们连接起来使用

    • 0：不使用
    • WNOHANG：没有子进程退出，它也会立即返回
    • WUNTRACED：跟踪调试
