git

git概念

  • 工作区：仓库的目录。工作区是独立于各个分支的。
  • 暂存区：数据暂时存放的区域，类似于工作区写入版本库前的缓存区。暂存区是独立于各个分支的。
  • 版本库：存放所有已经提交到本地仓库的代码版本。
  • 版本结构：树结构，树中每个节点代表一个代码版本。

注意有一个头指针HEAD指向当前所在结点

全局设置

  1. git config --global user.name xxx：设置全局用户名，信息记录在~/.gitconfig文件中
  2. git config --global user.email xxx@xxx.com：设置全局邮箱地址，信息记录在~/.gitconfig文件中
  3. git init：将当前目录配置成git仓库，信息记录在隐藏的.git文件夹中
  4. .gitignore：将此文件放在仓库根目录，里面编写的内容即为不希望归入git仓库管理的文件

常用命令

  1. git add XX ：将XX文件添加到暂存区
    • git add . ：将仓库目录下的所有已修改的文件添加到暂存区
  2. git commit -m "给自己看的备注信息"：将暂存区的内容提交到当前分支
  3. git status：查看仓库状态
  4. git log：查看当前分支的所有版本
  5. git reflog：查看HEAD指针的移动历史（包括被回滚的版本）
  6. git push -u (第一次需要-u以后不需要) ：将当前分支推送到远程仓库
    • git push origin branch_name：将branch_name分支推送到远程仓库
  7. git clone git@git.acwing.com:xxx/XXX.git：将远程仓库XXX下载到当前目录下
  8. git branch：查看所有分支和当前所处分支

查看命令

  1. git diff XX：查看XX文件相对于暂存区修改了哪些内容
  2. git status：查看仓库状态
  3. git log：查看当前分支的所有版本
  4. git log --pretty=oneline：用一行一个版本来显示
  5. git reflog：查看HEAD指针的移动历史（包括被回滚的版本）
  6. git branch：查看所有分支和当前所处分支
  7. git pull ：将远程仓库的当前分支与本地仓库的当前分支合并

删除命令

  1. git rm --cached XX：将文件从仓库文件列表和暂存区中删掉，不希望管理这个文件（相当于家族除名）
    • 本地仓库目录中有很多文件，不一定所有文件都成为仓库文件的一部分，即不是仓库文件的一部分则为没被管理的文件
  2. git restore --staged xx：将xx从暂存区里移除，但还管理此文件，删除后，此文件在暂存区中为当前分支节点上的最新版本
  3. git checkout — XX 或 git restore XX：将XX文件尚未加入暂存区的修改全部撤销，即回滚到暂存区里存储的版本

回滚版本

  1. git reset --hard HEAD^ 或 git reset --hard HEAD~ ：将代码库回滚到上一个版本
  2. git reset --hard HEAD^^：往上回滚两次，以此类推
  3. git reset --hard HEAD~100：往上回滚100个版本
  4. git reset --hard 版本号：回滚到某一特定版本，版本号在git log 中的前七位哈希值可看
  5. git checkout — XX 或 git restore XX：将XX文件尚未加入暂存区的修改全部撤销，即回滚到暂存区里存储的版本
  6. 私人回退远程仓库版本：
    1. 回退本地分支版本
    2. 强制推送到远程仓库：git push -f
  7. 公共回退远程仓库版本：

远程仓库

  1. git@git.acwing.com:xxx/XXX.git 第一个git是用户名，@后面到:是服务器IP或域名，xxx是仓库主人id，XXX是仓库名字
  2. git remote add origin git@git.acwing.com:xxx/XXX.git：将本地仓库关联到远程仓库
  3. git remote -v：查看本地仓库关联的远程仓库
  4. git remote remove origin：取消远程关联的仓库
  5. git clone git@git.acwing.com:xxx/XXX.git：将远程仓库XXX下载到当前目录下
  6. git push -u (第一次需要-u以后不需要) ：将当前分支推送到远程仓库
  7. git push -f 强制推送到远程仓库
  8. git push origin branch_name：将本地的branch_name分支推送到远程仓库
  9. git push --set-upstream origin branch_name：设置本地的branch_name分支对应远程仓库的branch_name分支
    • 相当于创建远程仓库的新分支
    • 其实也可以直接git push会有提示这条命令出现
  10. git push -d origin branch_name：删除远程仓库的branch_name分支
  11. git checkout -t origin/branch_name 将远程的branch_name分支拉取到本地
  12. git pull ：将远程仓库的当前分支与本地仓库的当前分支合并
    • 即：将远程仓库版本同步到本地（与push相反）
    • 相当于将云端的版本拿下来，然后merge一下
  13. git pull origin branch_name：将远程仓库的branch_name分支与本地仓库的当前分支合并
  14. git branch --set-upstream-to=origin/branch_name1 branch_name2：将远程的branch_name1分支与本地的branch_name2分支绑定对应
    1. 当本地没有此分支时，先手动创建branch_name2分支
    2. 其次再执行上述命令
    3. 注意，只是本地此分支与云端分支绑定对应，并没有把版本拉取下来，还需要git pull一下

分支命令

  1. git branch branch_name：创建新分支

  2. git branch：查看所有分支和当前所处分支

  3. git checkout -b branch_name：创建并切换到branch_name这个分支

  4. git checkout branch_name：切换到branch_name这个分支

  5. git merge branch_name：将分支branch_name合并到当前分支上

    • 如果遇到冲突时，即当前分支和想合并的分支都不同时，执行完merge操作后，会提示你需要手动合并的文件

      如：某个冲突文件中的某几行

      需要手动解决的就是处理冲突的内容，并且加到暂存区里，且需要commit一下

  6. git branch -d branch_name：删除本地仓库的branch_name分支

  7. git push --set-upstream origin branch_name：设置本地的branch_name分支与远程仓库的branch_name分支绑定对应

  8. git push -d origin branch_name：删除远程仓库的branch_name分支

  9. git checkout -t origin/branch_name 将远程的branch_name分支拉取到本地

  10. git pull ：将远程仓库的当前分支与本地仓库的当前分支合并

  11. git pull origin branch_name：将远程仓库的branch_name分支与本地仓库的当前分支合并

  12. git branch --set-upstream-to=origin/branch_name1 branch_name2：将远程的branch_name1分支与本地的branch_name2分支绑定对应

stash暂存

  1. git stash：将工作区和暂存区中尚未提交的修改存入栈中
  2. git stash apply：将栈顶存储的修改恢复到当前分支，但不删除栈顶元素
  3. git stash drop：删除栈顶存储的修改
  4. git stash pop：将栈顶存储的修改恢复到当前分支，同时删除栈顶元素
  5. git stash list：查看栈中所有元素

.gitignore文件

我们做每一个项目都需要用 git 来进行版本控制和管理，而.gitignore文件正如其名字一样，告诉 git 哪些文件不需要添加到版本管理中。

通常在匹配文件时，支持大部分正则表达式

如：

!：不过滤（放开头）

/：目录

  • 放开头：整个目录忽略
    • bin/：忽略当前路径下的bin文件夹，该文件夹下的所有内容都会被忽略，不忽略 bin 文件
  • 放末尾：目录下所有文件被忽略，目录本身不忽略
    • /bin：忽略根目录下的bin文件

*：多个字符的通配符

**：多级目录（可在开始，中间，结束）

?：通用匹配单个字符

[]：通用匹配单个字符列表

  • *.[oa]：忽略后缀名为 .o 和 .a 的文件
