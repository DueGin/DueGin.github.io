发表于 2022-04-16 | 分类于 C/C++

inline

使用 inline 修饰函数，被称为内联函数

像这样：

优点：

  • 方便重复调用
  • 提升运行效率

缺点：

  • 代码膨胀 –> 内存爆炸
  • 只能返回简单的函数体结构
  • 还得看不同编译器

所谓内联，就是在调用某个函数时，将此函数的函数体直接替换到函数调用的地方，函数实体直接嵌入在调用的地方，这一点类似宏，所以运行效率特别高，这样就不会存在函数跳转、出栈入栈的操作了。

  • 相当是空间换时间的策略

  • 只适合函数体简单的函数：但 inline 只适合函数体内代码简单的函数使用，一旦用上的循环结构，则代码体复制到调用的地方是，将会导致内存爆炸的问题。

  • 加上 inline 只是给编译器一个建议：使用 inline ，该函数也不一定成为内联函数，关键还得看编译器

extern

使用 extern 声明变量为外部的变量，且不进行初始化，可以使用其他文件中已经定义且赋值了的同名静态变量，这个称为单定义规则

extern仅允许本文件的全局变量创建外部链接

且只能在某一文件中定义一次，不能多次定义，其他文件用到此变量则需要用 extern 修饰即可

如：

但是要使用的话，直接运行t.cpp文件是不行的，需要链接t1.cpp文件才行

即：g++ t.cpp t1.cpp -o test 编译并链接且生成名为test的可执行文件

若在某一函数再次用 extern 声明此变量 x ，那么它将在此函数中使用外部变量 x 对其做出的修改也是同步到此变量初始化的原文件中的，再函数内再次声明的操作，可有可无。若直接定义一个同名变量，且初始化，则优先使用该局部变量，若想使用外部变量 x ，则使用 ::x 即可

noexecpt

不抛出异常，也不能乱用，因为很麻烦，有可能导致程序终止

当一个声明为noexcept的函数抛出异常时，程序会被终止并调用std::terminate()

constexptr

  • 用 constexptr 修饰函数

被编译器检测到为编译期常量，那么这个函数就可以自动地在编译期运行。

如果我们给一个函数加上一个constexpr关键字，不是说我们就把这个函数绑死在编译期上了，这个函数也应该能在运行期被复用。如果一次调用被认为是runtime的，那么这个函数返回的值也不再是编译期常量了——它就被当作一个正常的函数来对待

  • 用 constexptr 修饰的变量

声明时带有constexpr关键字的变量是常量表达式，因而可以被用做编译期计算。

我们可以使用 constexpr 构造函数来创造它的编译期常量对象，鉴于它也有 x 的 constepxr 类型的 getter 函数，我们也可以在编译期使用这些函数来获取它的成员值。

也就是说声明的变量需要与获取此变量值的来源函数也需要声明为 constexptr 的，我理解为一一对应关系

decltype

用于查询表达式的数据类型。主要是为泛型编程而设计，以解决泛型编程中，由于有些类型由模板参数决定，而难以（甚至不可能）表示之的问题。

它与auto类似，但是又有区别，在此过程中，编译器分析表达式并得到它的类型，却不实际计算表达式的值

形如：decltype(v1) v2

  • 若v1为变量时，decltype返回的结果为v1的类型

因为 b 和 *p 是一个引用，decltype(b) 和 decltype(*p) 的结果就是引用类型，因此作为引用的 z 和 c 必须被初始化。

切记：decltype((v1)) v2（注意是双层括号）的结果永远是引用，而decltype(v1) v2结果只有当v1本身就是一个引用时才是引用。
